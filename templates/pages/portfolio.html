Wiener Takes All

Wiener Takes All is a fast-paced, action-packed, wacky game of reverse tag where everyone is a hotdog bun and is trying to get the wiener. I made Wiener Takes All with a small team of other undergraduate students in a course called Games Programming. This project fulfilled my desire to create a full game without the use of a framework. In this game, we got to design many of the interesting components of a game, including the rendering and data architecture.

The two most complicated sections were the rendering and physics engines, done using OpenGL and PhysX respectively. The physics proved very difficult to tweak but I think the result we ended up with gave a good fun balance. We also used a data-oriented design architecture. While I was skeptical at first, a data-oriented design proved to be very nice to use when coupled with an intuitive event system.

This project gave me a strong appreciation for game engines and has better prepared me to use them effectively. I also gained hands-on experience in using matrices and other math for rendering concepts. Now I've moved onto creating larger games using actual a full game engine such as Unity, but I feel much better equipped in having this project under my belt. A packaged windows binary is available for download as a release on GitHub.

Solar Car Team Epsilon Embedded Systems

This project involved designing and writing all the software for the custom-built embedded systems going into the 5th Generation Solar Car. This was particularly exciting for me because it was the first time I really had a chance to work on actual embedded systems and all their intricacies. The picture may be slightly misleading as I didn't do any of the hardware design, but I designed and coded all the software. The hardware design was all overseen by the extremely talented Logan Hunter.

The software stack is built on top of the STM32 libraries. The code was first initialized using STM32CubeMX, then a Makefile generated using CubeMX2Makefile, then cross-compiled with gcc4mbed. We have a guide to use this stack if you so wish yourself.

On top having experience setting up an embedded systems development environment, I wrote a lot of code surrounding a CAN network, the industry standard protocol for communications within an automobile. I also learned a great deal about sharing resources in a multi-task environment with FreeRTOS. Overall, I now feel comfortable coding in pure C and working with low-level concepts such as registers and limited memory.

Terminal Fighter

Terminal Fighter is a typing game written in Python3. This was a passion project with a few friends that I pursued and is still in development. What I have shown below is the finished prototype, a finished product will be quite large in scope, I don't expect it to be done for a while. If you want to try it out you can run terminalfighter.py using Python3, you will need pygame which can be installed for Python3 with these instructions. Pres 1 and 2 to switch weapons!

I love typing games and I had always wanted to try making one as games are a unique development experience. This project gave me very good opportunities to challenge and practice many object-oriented concepts. This is due to the large number of objects that are similar yet different, and also must be constantly created and destroyed. I also was able to design a game loop and see how different designs affected the game in different ways. We also faced the bottleneck of SDL1 which did not take advantage of the graphics card, the next iteration is in C++ and using SDL2. This next step has me very excited to even further improve my object-oriented design intuition and also stretch the capabilities SDL2 has to offer.

Overall although the graphics are quite simple, I'm very happy with the gameplay result as it has very few bugs (that I've found anyways). Games, although commonly looked down upon in development, I think are extremely fun to develop and pose many difficult and unique challenges.

The project is built on the QT framework, from this project a large majority of the work in the Solar Car Team is now also done in QT. This project really kickstarted my love for software development and showed me what it really meant to design modular object-oriented software. I also realized from this project how much I loved making user interfaces.
